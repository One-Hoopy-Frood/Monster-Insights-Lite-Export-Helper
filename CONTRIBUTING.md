# Contributing

Thanks for helping improve MI Data Exports! This project is early and evolving.

## Getting Started
- Fork the repo and create a feature branch.
- Keep changes minimal and focused.
- Follow WordPress/PHP best practices: escape, sanitize, and validate.

## Development
- Place code in `includes/` and wire it from `mi-helper-reports.php`.
- Prefer prefixed functions/classes to avoid collisions.
- Write small, testable units of code.

## Commit & PR
- Use clear, descriptive commit messages.
- Reference issues when applicable.
- Open a PR; CI will run PHP syntax checks.

## Code Style
- PHP 7.4+ syntax.
- WordPress coding practices for hooks, escaping, and i18n.
